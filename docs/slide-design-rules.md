# スライド作成ルール

本ドキュメントは、Marp形式のスライドを作成・レビューする際に従うべきルールを定義する。

## 1. 1スライド＝1メッセージの原則

- 各スライドで伝えたいことは **1つだけ** に絞る
- スライド上部にそのスライドの **主張（キーメッセージ）** を見出しとして明記する
- 見出しを読むだけでストーリーが追えるようにする（見出しだけの流し読みテスト）
- メッセージが2つ以上ある場合はスライドを分割する

## 2. ストーリー設計

- スライド全体を通じたロジックの流れを先に設計してから個別スライドを作る
- 聞き手の視点で最適な順序に並べる（作り手の都合で並べない）
- 序盤に課題・背景を提示し、段階的に解決策や結論へ導く構成にする
- 各スライドの見出しをつなげて読んだとき、一貫したストーリーになっているか確認する

## 3. Less is More（S/N比の最大化）

- 不要な装飾・テキスト・要素を徹底的に削る
- 情報の **Signal（伝えたいこと）** を最大化し、**Noise（それ以外）** を最小化する
- 絵文字は最小限に留め、内容と直接関係するもののみ使用する
- 箇条書きは1スライドあたり **3〜5項目** を上限とする
- 長文を避け、キーワードや短いフレーズで表現する

## 4. 図解・データの視覚化

- 数値データはテキストではなく **適切なグラフ** で表現する
  - 比較 → 棒グラフ
  - 推移 → 折れ線グラフ
  - 構成比 → 円グラフ・帯グラフ
  - 変化の内訳 → 滝グラフ（ウォーターフォール）
- 概念や関係性は **チャート（概念図）** で表現する
  - マトリクス、フローチャート、ピラミッド構造、ベン図 等
- 表はセルを最小限にし、強調したい数値に色を付ける

## 5. レイアウトと視認性

- フォントサイズは十分な大きさを確保する（本文 `0.9em` 以上）
- コードブロックのフォントサイズは `0.7em〜0.8em` 程度に抑え、はみ出しを防ぐ
- 1スライドにコードブロックを入れる場合は **コード＋要点を1〜2行** に留める
- テキストがスライド下部やフッターに被らないよう余白を確保する
- 色は3色以内に抑える（ベース色・アクセント色・テキスト色）

## 6. 簡潔なタイトル設計

- タイトルスライドでは **主題＋副題** の構成にする
- 各スライドの見出し（`##`）は、そのスライドの主張を一言で表す
- 見出しに装飾的な言葉を入れず、内容を端的に伝える

## 7. エビデンスと出所

- 外部データやグラフを使う場合は出所を脚注に記載する
- 自チームの実績データであれば「実測値」「社内データ」等と明記する

## 8. Marp固有のルール

- `---` でスライドを区切る
- `paginate: true` でページ番号を表示する
- ヘッダー・フッターは簡潔に（プロジェクト名 / コピーライトのみ）
- テーマのカスタムCSSは最小限にし、デフォルトテーマの上に薄く被せる
- 画像エクスポート後に文字はみ出し・レイアウト崩れを必ず確認する

## レビュー時のチェックリスト

- [ ] 各スライドのメッセージは1つだけか
- [ ] 見出しだけ読んで全体のストーリーが通るか
- [ ] 不要な要素（装飾、冗長なテキスト、使われていない色）はないか
- [ ] 箇条書きが多すぎないか（3〜5項目以内）
- [ ] コードブロックがスライド領域をはみ出していないか
- [ ] 数値データを適切なグラフで表現できていないか
- [ ] テキストがフッターに被っていないか
